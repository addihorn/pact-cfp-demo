# Product Steering

## Purpose
Provide a compact demonstration of 3-tier system architecture using a Todo domain.

## Primary Audience
- Engineers learning service boundaries
- Stakeholders reviewing architecture decomposition

## Product Scope (v1)
- Single-tenant Todo CRUD with completion state and list filtering
- Operational endpoints for health/readiness/metrics
- Local-first developer workflow via Docker Compose

## Out of Scope (v1)
- Authentication and authorization
- Multi-tenant isolation
- External integrations
